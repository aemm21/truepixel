👁️ TruePixel: Open-Source AI Image Verification Engine

TruePixel is a lightweight, private, open-source engine designed to detect synthetic images generated by Artificial Intelligence (Deepfakes, Midjourney, DALL-E, Stable Diffusion).

Unlike commercial detectors that analyze superficial visual imperfections, TruePixel operates under a dual scientific approach: it analyzes pixel geometry and searches for physical anomalies in the Fourier frequency domain that are mathematically impossible for generative diffusion models to replicate seamlessly.

⚠️ CRITICAL NOTICE: TRAINING REQUIRED

🔴 IMPORTANT: TruePixel is an architectural framework. Out of the box, the model weights are initialized randomly.

You must train the model with your own dataset of real and AI-generated images before using verify.py or api.py. Running the model without training will result in random predictions (usually defaulting to "Suspicious" or "Real" because the untrained weights fall in the middle of the sigmoid distribution).

To make TruePixel functional, follow the Training Section below to build your truepixel_v1.pth weights.

🧠 Dual-Branch Architecture

TruePixel processes each image file through two independent parallel channels before emitting a unified probabilistic judgment:

               ┌─────────────────────────────┐
               │         Input Image         │
               └──────────────┬──────────────┘
                              │
              ┌───────────────┴───────────────┐
              ▼                               ▼
     [ Spatial Branch ]            [ Frequency Branch ]
  - Analyzes pixels/textures -    - Fourier Transform (FFT) -
  - 2D Convolution (RGB)     -    - Spectral Log-Magnitude  -
              │                               │
              └───────────────┬───────────────┘
                              ▼
                  [ Feature Concatenation ]
                              │
                              ▼
                    [ Sigmoid Classifier ]
                              │
                              ▼
                     📊 AI Probability %


Spatial Branch (Visual): A Convolutional Neural Network (CNN) optimized to identify texture artifacts, edge inconsistencies, and local color anomalies.

Frequency Branch (Spectral): Converts the image tensor using a 2D Fast Fourier Transform ($\text{FFT}_{2\text{D}}$). Images created by AI exhibit periodic patterns and geometric spikes in high frequencies due to mathematical upsampling and pixel interpolation.

📦 Repository Structure

The project maintains a clean, modular structure for easy extensibility:

File

Description

model.py

Architecture definition of the unified neural network (TruePixelNet).

Fourier magnitude spectrum core mathematical operations optimized in PyTorch.

🚀 Quick Start Guide

1. Clone and Install Dependencies

Ensure you have a virtual environment active, and execute:

# Clone the repository
git clone [https://github.com/aemm21/truepixel.git](https://github.com/aemm21/truepixel.git)
cd truepixel

# Install required dependencies
pip install -r requirements.txt


2. Parametric Training (CLI)

Before verifying any images, you need to train the network. Set up your dataset by placing real photos in ./data/reales and AI-generated images in ./data/sinteticas.

You can configure the training process directly from the terminal using arguments:

# Run training with custom hyperparameters
python train.py --epochs 40 --batch_size 16 --lr 0.0003 --val_split 0.20


Available Parameters:

--dir_reales: Path to the directory containing authentic photos (Default: ./data/reales).

--dir_sinteticas: Path to the directory containing synthetic AI images (Default: ./data/sinteticas).

--epochs: Number of passes through the complete dataset.

--lr: Learning rate for the Adam optimizer.

--val_split: Percentage of images reserved for validation (e.g., 0.20 = 20%).

--output: Name of the generated weights file (Default: truepixel_v1.pth).

🌐 Web API Deployment

Once trained, TruePixel includes a FastAPI server so you can query the model from external clients (mobile apps, browser extensions, or moderation backends):

# Start the local API server
uvicorn api:app --reload


Once the server is running, access the interactive documentation at:
👉 http://127.0.0.1:8000/docs

There, you can test the POST /verificar/ endpoint by uploading an image and receiving a structured JSON response:

{
  "archivo": "suspicious_sample.png",
  "probabilidad_ia_porcentaje": 91.43,
  "veredicto": "ALTA PROBABILIDAD DE DEEPFAKE / IA"
}
